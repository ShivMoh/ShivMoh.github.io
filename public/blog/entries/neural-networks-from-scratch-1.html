Things like regularisation and momentum were nice concepts to learn but I always like learning the bare minimum to get something working from there so that's what I did.

The neural network to recreate was as follows (code is pytorch): 

class SimplestMNIST(nn.Module):
    def __init__(self):
        super().__init__()
        self.flatten = nn.Flatten()
        self.dense = nn.Linear(784, 10)

    def forward(self, x):
        x = self.flatten(x)
        return self.dense(x)

Recreating it from scratch

The code needed to this was a lot less than I thought. In general constructing neural networks, while I'm sure theoretically very complex was quite straight forward once you get your head around it. The calculus and linear algebra needed is quite manageable and even if you don't understand it that well (like I do :), black boxing helps if you simply just treat it as a function of sorts.

Starting with the basic Linear layer, all we have to do is just construct a forward pass and a backward pass. The forward pass is just a matrix operation where dot the input matrix and the current weights for that layer. And the backward pass is just that in reverse (using the magic of derivitives.).

Next up was the flatten layer, which really and truly just took a batch of images in some shape and well, flattened it, meaning that instead of having an input of (64, 1, 28, 28), we get something like (64, 728). Now do I really understand how that remains mathematically sound? By that, I mean, how does it preserve all the previous information given the images, once in the shape of (1,28,28) are now simple (1, 728)? My answer to that is I don't really know. One day I might look into it more but for now, its sufficient for me to simply know that this puts the batch of images in a form that makes it convient for matrix multiplication with the weights of the neural network. So then our neural network can be in the shape of something like (728, 10) such that we can multiply the inputs by the weights to get the shape of (64, 10).

Next up was the basic and simpliest optimizer, Schothastic Gradient Descent (SGD). All this does is simple takes a random step in some direction where the gradient is negative (that is, where the loss is likely to be lower). Its very basic but it gets the job done. For the loss function we used the Cross Entropy loss which is good for classification. And that's about it (or atleast all I'm writing about it for now).
